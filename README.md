# MamaWorld Tests

A QA automation and testing project for validating the MamaWorld application. This repository is used to organize test assets, automate checks, document scenarios, and support reliable software delivery.

## Overview

`mamaworld-tests` contains testing resources for the MamaWorld project, including:
- Manual test cases.
- Automated test scripts.
- API test validation.
- Regression test coverage.
- Bug tracking and test documentation.

## Features

- Organized test cases and test scenarios.
- Support for manual and automated testing.
- API testing and response validation.
- Regression and smoke test coverage.
- Reusable test data and utilities.
- Clear documentation for test execution and maintenance.

## Project Structure

```text
mamaworld-tests/
├── tests/            # Test scripts and scenarios
├── test-data/        # Sample data and fixtures
├── reports/          # Test execution reports
├── docs/             # Supporting documentation
├── utils/            # Helper functions and shared utilities
└── README.md
```

## Requirements

Depending on the test framework used in this project, you may need:
- Python 3.10+ or Node.js 18+.
- A browser automation framework such as Playwright or Selenium.
- API testing tools such as Postman or a test runner.
- Git for version control.

## Getting Started

### 1. Clone the repository

```bash
git clone <repository-url>
cd mamaworld-tests
```

### 2. Install dependencies

If this project uses Python:

```bash
pip install -r requirements.txt
```

If this project uses Node.js:

```bash
npm install
```

### 3. Configure environment variables

Create a local environment file if needed:

```bash
cp .env.example .env
```

Update the values for your test environment.

### 4. Run tests

Examples:

```bash
pytest
```

or

```bash
npm test
```
